Hexo主题安装及配置

安装主题

从GitHub上克隆主题，在自己的目录根目录下右键点击Git Bash，输入以下命令 git clone schemes_download_url file_path （schemes_download_url：主题的下载地址，file_path：主题所在的项目目录）。我这里使用的是Next主题，大家可以根据自己的喜好去下载主题。

配置文件说明

在 Hexo 中有两份主要的配置文件，其名称都是 _config.yml。 其中，一份位于站点根目录下，主要包含 Hexo 本身的配置；另一份位于主题目录下，这份配置由主题作者提供，主要用于配置主题相关的选项。

为了描述方便，在以下说明中，将前者称为 站点配置文件， 后者称为 主题配置文件。

主题配置

主题配置文档在官方网站上已经讲得很详细了，在这里就不再赘述。
这里讲一下，为什么自己的文章都是直接展开的，而不是以摘录的形式显示在首页。

Hexo-设置阅读全文
下面提供三种设置阅读全文的方法：
1.在文章中使用手动进行截断
这种方法可以根据文章的内容，自己在合适的位置添加标签，使用灵活，也是Hexo推荐的方法

2.在文章中的 front-matter 中添加description，并提供文章摘录
这种方式只会在首页列表中显示文章的摘要内容，进入文章详情后不会再显示
3.自动形成摘要，在主题配置文件中添加
默认截取的长度为 150 字符，可以根据需要自行设定

PS：建议使用第一种方式，除了可以精确控制需要显示的摘录内容以外， 这种方式也可以让 Hexo 中的插件更好的识别。

添加评论模块

使用 GitHub Issues 搭建评论系统，注册OAuth Application，点击此处 注册一个Application。注册成功之后会分配一组client_id和client_secret，在接下来的配置中将会用到这两个值。配置成功之后登录GitHub并初始化评论即可。
具体设置，参照 文章

参数说明：

  • id：博客文章的文件名
  • title：博客文章的标题。一般情况下是不需要这个参数，增加这个参数是为了让id和title形成唯一标识，文章的评论会对应的归类，当这两个参数修改了的情况下，文章对应的评论也就不复存在，所以在配置的时候想好，不要轻易改动。
  • owner：GitHub用户名
  • repo：在GitHub上新建一个Comments仓库
